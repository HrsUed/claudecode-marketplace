# Phase 1: 現状確認

## 目的

3本柱の状態を把握し、リスクを評価する。

## 3本柱

| 柱 | 確認方法 | 判定 |
|----|----------|------|
| Version Control | `git status` | ✅管理済 / ❌未管理 |
| Testing | テストファイル有無 | ✅あり / ❌なし |
| Automation | CI設定有無 | ✅あり / ❌なし |

## 非決定的要素の検出

```php
// 検出すべきパターン
rand(), random_int()           // 乱数
time(), now(), new DateTime()  // 時刻
Str::uuid(), uniqid()          // UUID
Http::get(), curl_exec()       // 外部通信
```

## リスク評価

| リスク | 条件 |
|--------|------|
| 🔴 高 | テストなし + 複雑度高 + 外部依存多 |
| ⚠️ 中 | テスト不十分 または 複雑度高 |
| ✅ 低 | テストあり + 複雑度低 |
