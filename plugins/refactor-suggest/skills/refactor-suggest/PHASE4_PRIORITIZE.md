# Phase 4: 優先度評価

## 目的

提案に優先度を付け、実施順序を決定する。

## 優先度基準

| レベル | 条件 | 例 |
|--------|------|-----|
| 🔴 High | バグ温床/テスト不可/理解困難/均衡度≤3 | 100行超メソッド、5段ネスト、結合問題 |
| ⚠️ Medium | 保守性低下/品質悪化/均衡度4-6 | 30-100行、マジック値散在 |
| 💡 Low | 美的改善/将来の拡張性 | 軽微な命名、小構造改善 |

### 結合バランス評価（オプション）

提案に結合の観点を加える場合は、`/coupling-analysis` を実行し均衡度を評価:

| 均衡度(B) | 優先度への影響 |
|-----------|---------------|
| B ≤ 3 | 🔴に昇格（設計問題あり） |
| B = 4-6 | ⚠️を維持（要注意） |
| B ≥ 7 | 優先度に影響なし |

## 実施順序の原則

1. **依存関係**: 前提となる手法を先に
2. **リスク順**: High → Medium → Low
3. **小さく**: 成功体験を積む

### 典型的な順序

```
Extract Function（分割）
    ↓
Guard Clauses / Decompose（条件整理）
    ↓
Extract Class / Polymorphism（設計改善）
    ↓
Replace Magic Literal（仕上げ）
```
