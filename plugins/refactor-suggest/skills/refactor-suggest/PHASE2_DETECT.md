# Phase 2: コードの臭い検出

## 目的

改善が必要なパターン（臭い）を検出する。

## 検出パターン

### A. メソッド

| 臭い | パターン | 閾値 |
|------|----------|------|
| 長いメソッド | 行数過多 | 20行 |
| 長いパラメータ | 引数多数 | 4個 |
| 重複コード | 類似ブロック | 3行×2箇所 |
| フラグ引数 | `bool`で分岐 | - |

### B. 条件分岐

| 臭い | パターン | 閾値 |
|------|----------|------|
| ネスト | `if { if { if` | 3段 |
| 複雑な条件 | `&& \|\|` 連結 | 3個 |
| switch散在 | 同条件の複数switch | 2箇所 |
| 型コード分岐 | 文字列/数値で型判定 | - |

### C. データ

| 臭い | パターン |
|------|----------|
| プリミティブ執着 | 概念がstring/intのまま |
| データの塊 | 常にセットの引数群 |
| マジック値 | `0,1,-1,''`以外のリテラル |

### D. クラス

| 臭い | パターン | 閾値 |
|------|----------|------|
| 巨大クラス | 責務過多 | 500行/20メソッド |
| 特性の横恋慕 | 他クラスのデータ多用 | 3回連続 |
| メッセージ連鎖 | `a->b()->c()->d()` | 3段 |
| 中間者 | 委譲のみ | 50%以上 |

## 出力

```markdown

| # | 臭い | 場所 | 深刻度 |
|---|------|------|--------|

| 2 | ネスト | process():L30 | 🔴 |
| 3 | マジック値 | L55,L78 | ⚠️ |

### 詳細

#### 1. 長いメソッド: `calculate()`
- **場所**: L45-L120 (75行)
- **問題**: 複数責務の混在
- **影響**: テスト困難、変更影響大

**次**: Phase 3
```
