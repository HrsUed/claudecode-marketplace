# Phase 1: コード分析

## 目的

対象コードの構造を把握し、メトリクスを収集する。

## 手順

### 1. 対象特定

| 入力形式 | 実行 |
|----------|------|
| ファイル | 該当ファイルを読み込み |
| メソッド | クラス内の該当メソッドを特定 |
| ディレクトリ | `find [dir] -name "*.php"` |
| `recent` | `git diff HEAD~1 --name-only` |

### 2. メトリクス収集

| メトリクス | ⚠️警告 | 🔴危険 |
|-----------|--------|--------|
| クラス行数 | 200行 | 500行 |
| メソッド行数 | 20行 | 50行 |
| パラメータ数 | 4個 | 7個 |
| ネスト深度 | 3段 | 5段 |

### 3. 構造把握

- クラス名・メソッド一覧
- public / protected / private の分布
- 依存関係（use文、DI）

## 出力

```markdown
## Phase 1 完了: コード分析

**対象**: [ファイル名]
**行数**: N行 / **メソッド数**: M個

### メトリクス

| メソッド | 行数 | 引数 | 状態 |
|----------|------|------|------|
| calculate | 45 | 5 | 🔴 |
| validate | 18 | 2 | ✅ |

### 警告
- 🔴 `calculate()`: 45行（閾値20超）
- ⚠️ `process()`: 引数6個（閾値4超）

**次**: Phase 2
```
