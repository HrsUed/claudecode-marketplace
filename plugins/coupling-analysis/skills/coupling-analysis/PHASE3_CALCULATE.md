# Phase 3: 均衡度計算

## 目的

3次元スコアから均衡度を計算し、問題のある結合を特定する。

## 計算式

```
モジュール性 = |S - D| + 1
均衡度 = max(|S - D|, 10 - V) + 1
```

## 判定基準

| 均衡度 | 評価 | アクション |
|--------|------|------------|
| 7-10 | ✅ 優秀 | 現状維持 |
| 4-6 | ⚠️ 要注意 | 改善検討（信頼度80%+で報告） |
| 1-3 | 🔴 問題 | 早急に対処（信頼度70%+で報告） |

## 問題パターン

| パターン | 条件 | 症状 |
|----------|------|------|
| 分散モノリス | S高 & D遠 | 調整コスト大、変更が波及 |
| 過度な抽象化 | S低 & D近 | 認知負荷大、間接化が過剰 |
| 不安定依存 | V高 & S高 | コア変更で全体に影響 |

## 計算例

```
GoodDesign:
  S=2, D=8, V=7
  モジュール性 = |2-8| + 1 = 7 ✅
  均衡度 = max(6, 3) + 1 = 7 ✅

→ S低 & D遠: コントラクト結合で距離が適切に離れている
→ 良好な設計（弱い結合を遠くに配置）
```

```
ProblematicDesign:
  S=7, D=6, V=9
  モジュール性 = |7-6| + 1 = 2 🔴
  均衡度 = max(1, 1) + 1 = 2 🔴

→ S≈D: 強度と距離が近い（分散モノリス傾向）
→ 高変動性(V=9)のコアドメインで問題が深刻
→ 改善必須: インターフェース導入 or 距離を遠ざける
```
