# Phase 2: 3次元評価

## 目的

各依存関係のS（統合強度）、D（距離）、V（変動性）を評価する。

## 手順

### 1. 統合強度(S)の評価

詳細: [SCALE_STRENGTH.md](SCALE_STRENGTH.md)

**クイック判定**:

```
内部実装に依存? ─Yes→ 9-10
       │No
双方向依存? ─Yes→ 8-9
       │No
同一TX協調? ─Yes→ 6-7
       │No
順序依存? ─Yes→ 4-5
       │No
モデル共有? ─Yes→ 2-3
       │No
DTO/IF のみ → 1
```

### 2. 距離(D)の評価

詳細: [SCALE_DISTANCE.md](SCALE_DISTANCE.md)

**クイック判定**:

| 位置 | D |
|------|---|
| 同一クラス | 1 |
| 同一ディレクトリ | 2 |
| 同一名前空間 | 3-4 |
| 異なるBC | 5-6 |
| 異なるパッケージ | 7-8 |
| HTTP/外部 | 9-10 |

### 3. 変動性(V)の評価

詳細: [SCALE_VOLATILITY.md](SCALE_VOLATILITY.md)

**クイック判定**:

| サブドメイン | V |
|--------------|---|
| レガシー/外部 | 1-2 |
| 汎用/インフラ | 3-4 |
| 支援 | 5-6 |
| 重要 | 7-8 |
| コア | 9-10 |

### 4. 信頼度を付与

| 確信度 | 条件 |
|--------|------|
| 90%+ | 明確な根拠あり |
| 80% | 高い確度で判断可能 |
| 70% | 推測を含む |
| <70% | 報告しない |
